1.1 운영체제란

운영체제(OS, Operating System): 
컴퓨터나 스마트폰 같은 시스템에서 하드웨어와 소프트웨어 자원을 관리하는 시스템 소프트웨어

1.1.1 운영체제의 구성 요소 커널 • 하드웨어를초기화해사용할수있게하는운영체제의핵심부분 • 여러소프트웨어가운영체제에서잘작 동할수있도록메모리와프로세스를 관리하고, 네트워크를연결하는 주요 기능을제공


●장치드라이버 • 하드웨어장치를초기화하고운영체제와의통신을담당하는소프 트웨어 • 장치: 그래픽카드, 랜카드, 디스크드라이브, 마우스, 키보드등 컴퓨터에연결해사용하는모든기기


●파일시스템 • 파일(file): 정보를 저장하는 기본 단위 • 파일시스템(file system): HDD, SSD 등 저장 장치에 파일을 저장하고 관리하는소프트웨어 • 파일시스템은파일의이름, 크기, 저장위치를저장하고, 파일에대한 읽기/쓰기/실행을제어하며, 파일의권한을관리하고저장장치에대 한접근도제어

●네트워크시스템 • 네트워크드라이버(network driver): 유선 랜, 와이파이, 블루투스 등네트워크장치를초기화해사용할수있게하는장치드라이버 • 네트워크스택(network stack): 네트워크 장치를 통해 들어온 네트 워크트래픽을처리하는네트워크프로토콜을구현한소프트웨어 • 네트워크프로토콜(network protocol): 네트워크에서 데이터를 주 고받는데적용되는통신규약


1.1.2 운영체제의 역할 ●프로그램실행과멀티태스킹 • CPU(Central Processing Unit, 중앙 처리 장치): 컴퓨터의 두뇌, 프 로그램실행에필요한기본명령어를실행하는하드웨어 • 프로세스(process): 실행 중인 프로그램

• 프로세스스케줄링(process scheduling) 프로세스중하나를골라CPU가실행하는것 • 스케줄링알고리즘(scheduling algorithm) 어떤프로세스를선택할지결정하는방법 • 프로세스스케줄러(process scheduler) 스케줄링하는주체 • 멀티태스킹(multi-tasking) 운영체제가프로세스스케줄러를사용해여러프로세스를동시에처리 하는방식


●인터럽트처리 인터럽트(interrupt): 하드웨어가 소프트웨어에 보내는 신호 ●메모리관리- 커널이사용하는메모리, 하드웨어에할당된메모리등사용목적에따라구분- 메모리주소체계: 물리메모리주소(physical memory address), 가상 메모리 주소 (virtual memory address) 등으로 구분해 관리- 가용메모리가부족하다면메모리영역의일부를디스크로잠시옮겨놓고메모리사용- 스와핑(swapping): 디스크에 저장된 데이터를 메모리로 복구해 사용하는 것


●사용자인터페이스제공 • 사용자인터페이스(user interface) 사람이컴퓨터시스템을사용하게하는방법 • 그래픽사용자인터페이스(GUI, Graphical User Interface) 그래픽을통해사용자와상호작용 • 명령어기반인터페이스(CLI, Command Line Interface) 텍스트기반명령어를통해사용자와상호작용


1.2 리눅스 배포판 • 리눅스배포판(Linux distribution) 운영체제의목적에맞게리눅스커널에여러프로그램을묶어제공하는패키지 • 리눅스배포판의종류 1. 2. 3. 레드햇리눅스(RHEL, Red Hat Enterprise Linux): 대표적인 유료 리눅스 배포판 센토스(Centos): RHEL 소스 코드를 기반으로 하는 무료 라이선스 배포판 우분투(Ubuntu): 무료로 배포되는 대표적인 리눅스 배포판


1.3 리눅스의 쓰임과학습이유 1.3.1 어디에 쓰일까 1. 서버: IT 서비스를구성하기위한서버 2. 클라우드컴퓨팅: 클라우드서비스를구축하기위한백엔드 3. 임베디드시스템: PC나서버환경에비해제한적인자원을가진 경우 4. 모바일기기: 스마트폰이나태블릿등

=================================================

1. 쉘(shell) 
    

리눅스가 제공하는 명령어 기반 인터페이스(CLI) 

쉘이란 명령어로 사용자가 시스템과 소통하는 방식 

사람이 컴퓨터에게 명령(명령어)를 내리고, 그 결과를 모니터로 확인한다. 

  

2. 터미널
    

컴퓨터와 사용자 간에 상호작용할 수 있게 해주는 연결 장치 

           사용자가 컴퓨터에게 명령을 전달하는 장치 => 입력장치 

           컴퓨터가 사용자에게 결과를 보여주는 장치 => 출력장치 

  

3. 셀(Shell)  CLI 프로그램 
    

사용자는 Shell을 통해 운영체제에게 명령을 내리고 결과 확인한다.

1971년에 켄 톰프슨 처음 개발 (유닉스 용 셸) 

1977년 본 셸 …. Csh Ksh Ash …..

=> Telent 과 네트워크 프로토콜을 이용하여 운영체제 터미널을 이용하여  접속 (원격 접속)

  

local     ⇔ remote 

  

4. 셸 스크립트 :  셸에서 동작가능한 명령어를 모아놓은 파일 
    

                                 여러 명령어를 한 파일에 모아 실행하는 방식 

  

#!/bin/bash  ⇒ 

/bin/sh (기본 셸) => 리눅스는 bash 가 기본쉘이다.  

셔뱅  shabang (스크립트를 처리할 인터프리터를 지정하는 역할)

스크립트 파일 첫 줄에 사용하는 특수한 문자열**


기본 명령어 

프롬프트 (prompt) : 쉘에서 사용자에게 입력을 받는 부분에 표시되는 텍스트 


-pwd : 

파일의 위치 정보 (현재 작업 디렉토리를 절대 경로로 조회하여 알려준다.)

          위치정보 : 상대경로  : 현재 작업하는 디렉토리를 기준인 파일 경로

                          절대경로  : 루트 디렉토리를 기준인 파일 경로 

- cat : 파일의 내용을 연결(화면에 출력)하거나 조회합니다. 
    

- exit  : 종료 명령어

 루트디렉터리 : 파일시스템의 최상단 위치 디렉터리    “/” 표기

/lib =>  시스템 라이브러리 설치
