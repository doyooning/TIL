# 📘 DOM 완전 정복 – 01. HTML만으로는 부족했던 이유

## 1. HTML의 본질과 구조적 한계

HTML(HyperText Markup Language)은 **하이퍼텍스트 문서를 표현하기 위한 마크업 언어**입니다. 프로그래밍 언어가 아니며, 복잡한 논리나 제어 흐름을 처리할 수 없습니다.

```
<h1>안녕하세요</h1>
<p>이것은 문단입니다.</p>
```

- 사용자의 행동에 반응하지 않음
- 종이에 인쇄된 책처럼 읽기 기능만 존재
- 버튼 클릭, 입력 반응, 동적 전환 등 상호작용 불가능

## 2. 상호작용의 요구와 자바스크립트의 등장

현대 웹은 단순 문서가 아닌 **실시간으로 변화하는 소프트웨어 플랫폼**입니다.

예시:

- 버튼 클릭 → 팝업
- 댓글 작성 → 화면 반영
- 마우스 오버 → 애니메이션
- 로그인 상태에 따라 메뉴 변경

이러한 기능은 **자바스크립트**로 구현됩니다.

## 3. DOM의 등장 – 문서를 객체로 바꾼 모델

자바스크립트는 HTML 파일을 직접 수정하지 않고, 브라우저가 HTML을 해석해 메모리 상에 구성한 **객체 구조(DOM)**를 조작합니다.

### DOM이란?

- **Document**: 문서
- **Object**: 객체
- **Model**: 구조화된 표현

### DOM 트리 구조 예시

```
<body>
  <div>
    <p>Hello</p>
  </div>
</body>
```

```
Document
└── body
    └── div
        └── p
            └── "Hello"
```

## 4. 트리 구조란?

HTML은 들여쓰기된 문장이 아니라, **부모-자식 요소가 중첩된 계층 구조**입니다.

### 현실 세계 예시: 조직도

```
회사
└── 부서
    └── 팀
        └── 팀원
```

## 5. DOM 조작 예제

```
<button id="btn">눌러보세요</button>
<p id="text">안녕하세요</p>

<script>
  const button = document.getElementById("btn");
  const text = document.getElementById("text");

  button.addEventListener("click", () => {
    text.textContent = "DOM을 통해 바뀌었어요!";
  });
</script>
```

### 작동 흐름

1. 브라우저가 HTML을 읽고 DOM 트리 생성
2. JS가 `getElementById()`로 DOM 요소 가져옴
3. 버튼에 클릭 이벤트 리스너 추가
4. 클릭 시 `.textContent` 변경
5. 화면 내용 즉시 변경

⚠ HTML 파일을 수정하는 것이 아니라, **브라우저 메모리 상의 DOM 객체를 실시간으로 조작**한 결과입니다.

## 6. DOM이 없었다면?

- JS는 HTML 요소를 찾을 수 없음
- 사용자 동작에 반응 불가
- 웹페이지는 항상 정적인 상태로만 존재

## 7. DOM 핵심 요약

|구분|설명|
|---|---|
|HTML|구조를 정의한 설계도|
|DOM|구조를 객체화한 메모리 트리|
|JS|객체 트리를 조작하여 웹을 움직이게 만드는 제어자|

## 8. 시각적 구조 요약

```
HTML 문서
├── 구조만 존재 (읽기 전용)
└── 클릭, 입력 등 반응 없음

브라우저 → DOM 생성
├── 객체로 구조화된 메모리 트리
└── 부모-자식 계층으로 연결

자바스크립트
├── DOM 객체 탐색 (getElementById 등)
├── 속성 수정 (textContent, style 등)
└── 이벤트 처리 (addEventListener 등)
```

## 9. 핵심 개념 정리 퀴즈

**Q1. 자바스크립트는 HTML 파일을 직접 수정할 수 있는가?** → ❌ 아니다. DOM 객체를 조작한다.

**Q2. DOM은 왜 필요한가?** → HTML을 JS가 다룰 수 있게 객체로 바꿔주는 브라우저의 메모리 구조이기 때문.

**Q3. DOM 트리는 무엇인가?** → HTML 요소들을 객체로 변환한 후, 부모-자식 관계로 연결한 계층 구조이다.

## ✅ 마무리 요약

HTML은 **구조를 정의한 문서**에 불과합니다. 이 정적인 문서를 자바스크립트가 조작할 수 있도록 **객체화한 것이 DOM**이며, DOM 덕분에 웹은 **동적인 반응을 구현**할 수 있게 되었습니다.

DOM을 이해한다는 것은 곧 **현대 웹 개발의 기본 언어를 습득하는 것**과 같습니다.
