#MySQL 

--------
##### 데이터 모델
현실세계의 정보들을 컴퓨터에서 표현하기 위해서 단순화, 추상화하여 체계적으로 표현하는 모델
데이터, 데이터 관계, 데이터의 의미와 일관성, 제약조건 등을 상세히 기술하기 위해 모델링 도구를 통해서 나온 결과물

##### 데이터 모델의 종류
- 관계형 데이터 모델(RDB)
	테이블(=릴레이션) : 최소 0개 이상의 행 + 1개 이상의 열
	열(Column, 속성) : 테이블 항목
		(속성의 개수 = 카디널리티)
	행(Row, 튜플) : 관련된 열의 집합
		(튜플의 개수 = 디그리)

##### 데이터 모델링 프로세스
1. 요구사항 분석
	사용자 및 시스템의 요구사항 정의 및 분석 단계
	요구사항 분석서, 인터뷰 기록 문서 등 산출물 작성
	
2. 개념적 데이터 모델링
	중요 개념을 구분
	==Entity와 관계 위주의 모델링으로 전반적인 골격 파악 중점==
	핵심 Entity 도출, ERD 작성
	-> DBMS 선정
	
3. 논리적 데이터 모델링
	각 개념을 구체화
	개념적 스키마(개체, 속성, 식별자, 관계) ->
	==논리적 스키마(릴레이션, 속성, 기본키, 외래키)==
	ERD-RDB 모델 사상(맵핑), 상세 속성 정의, 정규화 등
	
4. 물리적 데이터 모델링
	DB 생성 계획에 따라 개체, 인덱스 등 생성
	DB 개체 정의, 테이블 및 인덱스 등 설계
	-> 실질적인 테이블 생성 및 속성 기입 단계
	
5. 구현 및 유지보수
	
6. 검증 및 최적화
	
7. 문서화

##### ER 모델
세상의 사물을 개체(Entity)와 개체 간의 관계(Relationship)로 표현

###### ER 다이어그램
개체와 개체 간의 관계를 표준화된 그림으로 나타냄

##### 개체(Entity)
사람, 사물, 장소 등 유무형의 정보를 가지고 있는 독립적인 실체
-> 직사각형으로 표현

개체 타입의 유형
- 강한 개체 : 독자적으로 존재할 수 있는 개체
- 약한 개체 : 독자적으로 존재 불가, 상위 개체 타입을 가지는 개체

##### 속성(Attribute)
개체가 가진 성질
-> 타원으로 표현

###### 속성 유형
키 속성 -> 속성 이름에 밑줄
약한 개체 식별자 -> 식별자 아래에 점선
다중값 속성 -> 이중 타원
유도 속성 -> 점선 타원
복합 속성 -> 큰 타원 아래 작은 타원

##### 관계(Relationship)
개체 사이의 연관성을 나타내는 개념
-> 마름모로 표현

###### 관계 타입 
: 개체 타입과 개체 타입 간 연결 가능한 관계를 정의
- 차수(degree)에 따른 유형
	1. 1진 관계(자기 자신과 관계)
	2. 2진 관계
	3. 3진 관계
- 관계 대응수(cardinality)에 따른 유형
	1. 1:1 관계
	2. 1:多 관계
	3. 多:1 관계
	4. 多:多 관계
	`관계실선 위에 (최솟값, 최댓값) 표기, * : 여러 개 선택`
- ISA(is-a) 관계
	: 상위 개체 타입의 특성에 따라 하위 개체 타입이 결정
- 참여 제약 조건
	: 개체가 관계에 참여하는지 유무에 따라 구분
	1. 전체 참여 : 실선 2개 연결
	2. 부분 참여 : 실선 1개 연결

IE 표기법
: ERD를 더 축약하여 표현 -> 직사각형으로 표현

- 비식별자 관계(강한 관계)는 점선으로 표기
	-> 대부분은 비식별자 관계
	-> 한 개체의 PK가 다른 한 개체의 PK가 아니면서 FK

- 식별자 관계(약한 관계)는 실선으로 표기
	-> 종속적인 관계가 해당
	-> 종속되는 개체의 PK가 FK



관계 집합 : 관계로 연결된 집합을 의미


<데이터 모델링 예시 >
Entity
1 학과(학과번호, 학과명, 학과위치, 학과전화번호)
학과는 학과번호로 구분

2 학생(학번, 이름, 연락처)
학생은 학번으로 구분

3 과목(과목번호, 과목명, 학점)
과목은 과목번호로 구분

관계
학과 - 소속 - 학생 => one to many(1:N)
하나의 학과에는 여러 명의 학생들이 소속되어 있다.
(단, 신생학과는 소속학생이 없다.)
학생은 1개의 학과에만 소속되어야 한다.

학생 - 수강신청 - 과목 => many to many(M:N)
여러 명의 학생들은 여러 가지 과목들을 수강신청할 수 있다.
수강신청은 수강 가능한 기간이 정해져 있다.
학생은 수강신청할 과목을 정할 때 과목번호로 과목을 구분하여 신청한다.
학생은 신청한 과목에 대한 학점이 산출된다.

과목은 여러 명의 학생들이 신청할 수 있다.
과목은 학생들의 학점을 산출하여 보관한다.
과목은 과목번호에 의해 구분된다.
이는 과목명이 같더라도 구분할 수 있음을 의미한다.

결론 : 
학생 릴레이션과 수강신청 릴레이션 사이의 관계에서
학번은 수강신청의 일부,
수강신청을 고유하게 식별하는 데 반드시 필요하므로 식별관계로 정함.
과목번호도 학생이 수강신청한 과목을 구분하는 데 필요하므로 식별관계로 정함.

학과와 학생관계에서는 학과번호가 학생

관계형 데이터베이스

 데이터 삽입, 삭제, 수정할 때 불일치 또는 오류가 발생
=> 이상 현상
1. 삽입 이상
2. 삭제 이상
3. 수정 이상
-> 정규화를 통해 해결

정규화
: 데이터의 중복성과 종속성을 줄이도록 테이블을 구성하는 데이터베이스 설계 기술

큰 테이블을 작은 테이블로 분해하고
관계를 사용하여 관련된 테이블을 연결
